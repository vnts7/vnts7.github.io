State Management Quiz

  1. Can post requests be bookmarked? What are the problems?

    No. The bookmark does not save the request body

  2. What is the purpose of request dispatching?

    URL rewriting

  3. What is the difference between redirect and request dispatch?

    Redirect happens in client, dispatch happens in server

  4. What is an attribute ?

    An object bound into one of the three servlet API objects: HttpServletRequest, HttpSession, ServletContext

  5. What is the difference between attributes and parameters?

    Attribute is object and store in server, parameter is String and is sent from user

  6. What are dangers of using attributes?

    Application level attributes are not thread-safe

  7. What does it mean to say that http is stateless? Give an example of a stateful protocol.

    It means that after a web server responds to a request, there is no "memory" of the request or who sent it

    an example of a stateful protocol is TCP

  8. Give 5 different methods for maintaining state information (count each attribute scope as one method).

    request, session, application, persistent storage, cookies, hidden fields

  9. How long does a session last?

    It depends on how the server's configuration, can be configed via: session timeout, setMaxInactiveInterval or invalidate

  10. What is a cookie, and how long does a cookie last?

    Cookie is a piece of information that store in browser and will be sent back to sever whenever a request is made

    Can be temporary that browser removes when it closes OR

    Permanent that has a max age set

  11. What is the purpose of URL rewriting?

    User-friendly URL

  12. Why does the request attribute report "null" for the maintaining state demo?

    Because request attribute is short term computed results to pass from one servlet to another, when server finish responds to a request, request attributes are destroyed
